# 774converter(名無しコンバーター)

「ChatGPTやClaudeに相談したいけど、この文章、名前も住所も電話番号も入ってるんだよな…」

そのたびに手作業で伏せ字にして、返ってきた答えをまた手作業で戻して──面倒ですよね。
**774converter** は、その置き換え作業をまるごと引き受けるローカルアプリです。

## こんなふうに変わります

**Before(元の文章)**

> お世話になっております。株式会社山田製作所の田中太郎です。
> 担当の佐藤花子さんに引き継ぎます。連絡先: 090-1234-5678

**After(AIに渡す文章)**

> お世話になっております。会社AのAさんです。
> 担当のBさんに引き継ぎます。連絡先: 電話番号1

ラベルは「Aさん」「住所1」のような**人が読んでパッと分かる形式**。
AIの答えが返ってきたら、ワンクリックで全部元の名前に戻ります。

## 特徴

- 🏠 **完全ローカル動作** — 外部への通信は一切なし。検出も置換もぜんぶ自分のPCの中
- 🔁 **確実に元に戻せる** — 置き換えの対応表がJSONで手元に残り、逆変換はワンクリック
- 👀 **置き換え前に確認できる** — 検出結果を一覧でチェック。誤検出は外す、漏れは手動で追加
- 🤖 **LLM不要** — 検出は正規表現+GiNZA(日本語の固有表現抽出)。APIキーも課金もいりません

## 使い方

1. アプリを起動(下のセットアップ参照)→ ブラウザが自動で開きます
2. **「匿名化する」タブ**: 文章を貼り付け(.txt / .md / .csv のドロップもOK)→「個人情報を検出」
3. 検出結果を確認して「置き換える」→ できあがった文章をコピーしてAIへ
4. **「元に戻す」タブ**: AIの答えを貼り付けて「元の名前に戻す」→ 完成 🎉

## セットアップ

Python 3.10以上が必要です。初回のみ:

```bash
git clone https://github.com/m4n4c/774converter.git
cd 774converter
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

起動はMacなら `起動.command` をダブルクリック。またはターミナルで:

```bash
.venv/bin/uvicorn app:app --port 8756
```

ブラウザで http://localhost:8756 を開けば準備完了です。
※ 初回の検出は日本語モデルの読み込みで数十秒かかることがあります。2回目からはサクサク動きます。

## 置き換えラベルの一覧

| 種類 | ラベル例 |
|---|---|
| 人名 | Aさん、Bさん、…(敬称ごと置換: 「佐藤花子さん」→「Bさん」) |
| 組織 | 会社A、会社B、… |
| 住所 | 住所1、住所2、… |
| 電話番号 | 電話番号1、… |
| メール | メール1、… |
| 郵便番号 | 郵便番号1、… |

- 同じ人・同じ番号には、文章のどこに出てきても同じラベルが付きます
- 元の文章に「Aさん」という表現がすでにある場合は、衝突を避けて次のラベルを使います
- 復元は長いラベルから処理するので、「住所10」と「住所1」を取り違えません

## 安心して使うために

- 📁 **対応表(`mappings/`フォルダのJSON)が復元の鍵です。** これは自分のPCにだけ保存され、
  絶対にAIや外部サービスに送らないでください(このリポジトリにもコミットされない設定です)
- 🔍 人名の自動検出は高精度ですが100%ではありません。**置き換え前の確認画面で目視チェック**が
  いちばんの品質保証です。漏れを見つけたらその場で手動追加できます
- 同じ人でも表記が違うと別ラベルになります(「田中太郎」→Aさん、「田中さん」→Cさん)。
  対応表を見れば同一人物だと分かります

## しくみ(開発者向け)

- [app.py](app.py) — FastAPI サーバー(API + 画面配信)
- [detector.py](detector.py) — 個人情報の検出(正規表現 + GiNZA)
- [anonymizer.py](anonymizer.py) — ラベル割り当て・置換・復元・対応表の保存
- [static/index.html](static/index.html) — ブラウザUI(依存なしのvanilla JS)

GiNZA は confection 1.3系との互換性問題があるため、読み込み時に
`split_mode` を明示指定しています(detector.py 参照)。
