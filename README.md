# 774converter(名無しコンバーター)

テキストに含まれる個人情報(氏名・住所・電話番号・メールアドレスなど)を、
人間に読みやすいラベル(「Aさん」「住所1」「電話番号1」など)に置き換え、
あとで**確実に元に戻せる**ローカルアプリ。

外部への通信は一切行わない。検出は正規表現と GiNZA(日本語固有表現抽出)で、
LLM は使わない。

## 使い方

1. `起動.command` をダブルクリック(またはターミナルで `.venv/bin/uvicorn app:app --port 8756`)
2. ブラウザで http://localhost:8756 が開く
3. **匿名化する**タブ: テキストを貼り付け(またはファイルをドロップ)→「個人情報を検出」
4. 検出結果を確認。誤検出はチェックを外し、漏れは手動追加
5. 「チェックした項目を置き換える」→ 匿名化テキストをコピーして外部AIなどで処理
6. **元に戻す**タブ: 対応表を選び、処理済みテキストを貼り付け →「元の名前に戻す」

## ラベルの形式

| 種類 | ラベル例 |
|---|---|
| 人名 | Aさん、Bさん、…(敬称ごと置換: 「佐藤花子さん」→「Bさん」) |
| 組織 | 会社A、会社B、… |
| 住所 | 住所1、住所2、… |
| 電話番号 | 電話番号1、… |
| メール | メール1、… |
| 郵便番号 | 郵便番号1、… |

- 同じ文字列には常に同じラベルが付く
- 元テキストに同じ表現が既にある場合は衝突を避けて次のラベルを使う
- 復元は長いラベルから逆置換(「住所10」と「住所1」を取り違えない)

## 対応表(マッピング)

置換のたびに `mappings/mapping_日時.json` としてこのPCにのみ保存される。
**このファイルが復元の鍵**なので、匿名化テキストと一緒に外部へ送らないこと。

## 注意事項

- 人名検出(GiNZA)は 100% ではない。**置換前の確認画面で必ず目視チェック**すること
- 同一人物でも表記が違えば別ラベルになる(「田中太郎」→Aさん、「田中さん」→Cさん)。
  対応表を見れば同一人物と分かる
- ファイル読み込みは UTF-8 のテキスト系(.txt / .md / .csv / .log)に対応

## セットアップ(初回のみ)

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

## 構成

- `app.py` — FastAPI サーバー(API + 画面配信)
- `detector.py` — 個人情報の検出(正規表現 + GiNZA)
- `anonymizer.py` — ラベル割り当て・置換・復元・対応表の保存
- `static/index.html` — ブラウザUI
- `mappings/` — 対応表の保存先(自動生成)

※ GiNZA は読み込み時に `split_mode` を明示指定している(confection 1.3系との
互換性問題の回避。`detector.py` 参照)。
